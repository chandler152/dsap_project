# 山林接駁優化

## Proposal Report


### 動機與目標
台灣山林美景多集中在開車可達的熱門區，主因是大眾運輸不靈活，且密集的彎道與坡度對像我這樣易暈車的人來說是極大的心理障礙。目前的接駁系統缺乏對「路徑曲折度」與「遊客體質」的考量，導致許多秘境因交通「斷點」成為旅遊孤島。本計畫核心在於建立一套小巴路網優化模型，不僅量化路段的身體負擔，更著重於有效串聯多個潛在站點。透過重新規劃站點階級與接駁路徑，我們希望打破景點間的孤立狀態，建立一個具備包容性且高效的山林旅遊網絡，讓高質感的觀光不再受限於生理負荷。

### 預期功能
本計畫預期建立一套「動態路網診斷與優化模型」，將台灣山區地理環境轉化為具備多維權重的加權圖。系統核心功能在於量化路徑對遊客造成的身體負擔，透過整合距離、曲折度與坡度等關鍵參數，建立一套能反映真實旅遊痛點的評價體系。
在模擬分析層面，模型將引入遊客行為指標，模擬不同暈車耐受度的族群在面對大型巴士與小型接駁工具時的選擇邏輯。系統的主要任務是偵測並診斷「高吸引力但高交通阻力」的冷門節點，並進一步評定路線開發的經濟與社會價值。透過比較「投入接駁成本」與「提升之可達性與遊客流量」，模型能判斷特定路徑是否具備開發效益，進而針對路網斷點提供站點優化建議。最終，模型將輸出優化後的路線與站點規劃建議，證明透過彈性的低負擔運輸（如路徑靈活的小巴），能實質提升冷門景點的可達性，為台灣推廣兼具效益與包容性的旅遊方案。

### 使用技術
本計畫選用 C++ 作為核心開發語言。


### 時程規劃
第一週：基礎環境與小巴路網建模
定義站點（Station）與路段（Road）的類別設計，實作鄰接串列以儲存具備曲折度與坡度參數的圖形結構，並完成基礎測試資料的記憶體儲存。

第二週：核心路徑演算法實作
開發路徑搜尋邏輯，實作 Dijkstra 演算法並導入「暈車感權重」公式，產出起點至終點間最適合小巴行駛且體感舒適的最適路徑。

第三週：乘客群體建模與平均阻力運算
建立乘客群體（Passenger Group）模型，研究如何定義「平均舒適度」指標。系統需能在給定一組乘客構成的情況下，計算出該小巴路線的群體阻力分數，並輸出最能兼顧多數乘客體感的路徑。

第四週：原型整合與文字介面開發
完成 Console UI 開發，整合路網運算邏輯，讓使用者能透過交互輸入獲得小巴路徑規劃與預估負擔分數，正式提交 Prototype 驗證核心可行性。

第五週：路網斷點診斷與接駁優化
計算各景點在引入小巴接駁前後的阻力降幅，找出改善效益最大的交通斷點，並嘗試導入最小生成樹（MST）概念規劃最精簡的接駁網絡。

第六週：秘境開發價值評估系統
建立成本效益評判邏輯，結合景點吸引力與潛在流量增量，開發自動化標註功能，判斷特定冷門路段是否具備投入小巴接駁的經濟效益。

第七週：系統壓力測試與穩定度優化
利用較大規模虛擬資料進行路網模擬與壓力測試，檢查記憶體管理（Memory Leak）狀況，確保演算法執行效率並完成最終開發報告。

### 與課程的關聯
可能會使用的資料結構包含Adjacency List, Priority Queue。而使用的演算法可能有Dijkstra's Algorithm(除了路徑還要考慮曲折度)，Minimum Spanning Tree來分析如何以較少的路線成本串接冷門景點。
